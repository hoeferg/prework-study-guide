HTML

  • The parts of an HTML document are:
    • The head element contains information about the webpage.
    • The body element represents the visible content shown to the user.
    • The code !DOCTYPE html is a declaration that tells the browser wgat document type it should expect when opened.
    • The code html is th opening tag that start the start of the document. It will never be in the closing element.
    • The code lang="eg" is the language that the document is in.
    • Semaantic HTML elements are the name of the element that describes its content and the role it plays on the webpage.
  • Inside the head element can be the following:
    • A style element defines the css styles associated with the website.
    • A base element defines the base URL of a webpage. For example www.thewebsite.com is the base.
    • A link element connects an external resourse to the html document.
    • A meta element defines metadata. Metadata is the description of the website.
    • A script element embeds client side scripts.
    • A charset element sets the charater encoding.
  • Inside the body element can be the following:
    • A child element is an element that is nested inside other elements.
    • A div element creates a division on the page.
    • A span element creates an inline division.
    • An element is the entire element from opening to closing tag
    • An opening tag is the start of the code that creates the content.
    • A closing tag is the end of the code.
    • A heading element represents the heading a given text block represents.
    • A p element in HTML represents a paragraph.
    • A ul element in HTML represens an unordered or bullited list.
    • A ol element shows and ordered list.
    • A li element list content.
    • A img element creates an image.
    • A sre element shows the location of the image.
    • A br element creates a line of empty space.
    • An a element is an anchor that creates to links to webpages

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the contetn inside an element.
  • Incline CSS uses style attributes to style a particular page element. It's good for quick and permanent changes.
  • Internal style sheet applies style attributes to a page or multiple elements in the code.
  • External CSS style sheet is most common. It allows developers to keep all their CSS rules in a seperate file.
  • CSS box model going from outside in: margin, border, padding, content.
  • You can combine 2 styke elements with a comma.
  • Do Not Repeat Yourself (DRY): a coding principle that encourages developers to reuse or share code to reduce the number of lines of code.
  • The link attribute creates the connection to an external file.
  • The rek attribute specifies the relationship between the curremt document and linked document.
  • The href attribute specifies the location (URL)
  • Class attributes allow us to share a CSS rule to any ele,emt we choose by assigning the rule to a class attribute with a class selector.

    • git checkout: switch to another branch and check it out into your working directory.
    • git merge [branch]: merge the specified branch's history into your current one.
    • git log: show all tje commits in teh current branch's history.
    • git config -- global core.excludesfile [file]: system wide ignore pattern for all local repositories.
  • Git Vocabulary
